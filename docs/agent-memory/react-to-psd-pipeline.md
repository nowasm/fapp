Two capabilities to add to fapp, both **inside this repo** (decided 2026-06-16, user rejected separate-repo approach):

1. **React/HTML → canvas.json** (vector design稿). Must use runtime rendering (Playwright headless Chromium + DOM/getComputedStyle), NOT static JSX parse — oklch/clip-path/flex/web-components only compute in a browser. fapp is C++/python with no browser, so this lands as a new in-repo Node tool `tools/web2canvas/`. Output canvas.json feeds the native pipeline (figmaplay --shot self-verify, figmaedit, fapp2psd). React component boundaries → COMPONENT/INSTANCE = the naming/dedup spine.

2. **canvas.json → engine target, decided as direct godot .tscn** (NOT PSD — reversed 2026-06-16). PSD was dropped after discussion: PSD is a lossy raster handoff format (pixels+groups+names) that THROWS AWAY exactly the prefab semantics we want — 9-slice/cornerRadius, text-as-text, component identity, anchors/autoLayout, vector geometry. Going canvas.json→PSD→tscn would flatten then re-infer structure we already have. We control both ends, so PSD has no role. The html_ui_export reference project targets godot_ready/ (.tscn) directly, never mentions PSD. PSD is now only a possible OPTIONAL future sink (human Photoshop retouch, or Unity PSB/Cocos importers).
   **Decided: native C++ in fapp.** canvas.json (single source of truth) → `.tscn` + PNG sprite atlas + manifest.json. Reuse: src/canvas_parser.cpp, `Renderer::renderOverlay(nodes,…)` (renders a node subtree to transparent RGBA — pixel-identical to runtime, used to bake PNG sprites/9-slice textures), registerFontsFromDirectory/setImageDirectory, third_party/nlohmann. To build: layer-tree walker, content-hash dedup→shared sprites, semantic naming (from component identity), 9-slice inference (corner/cornerRadius → patch_margin/NinePatchRect), tscn text emitter (Label/RichTextLabel) + Control anchors, and a small extension to renderOverlay to render at native size into a tight buffer. **The PSD-writer risk is gone — .tscn is plain text, far easier than a PSD encoder.** Rasterization only emits PNG (godot consumes .tscn + .png, never a PSD container).

Shared spine: canvas.json is the IR already read by figmalib (C++) and fig2psd (TS). fig2psd's TS code is a reference impl (sanitizeName, dedup, 9-slice, PSD writing).
