# figo — 品牌资产

> 把 Figma 文件当 UI 运行时:**设计(.fig) + 逻辑(.js) = 会跑的 app**。

## 概念 — 一颗剖开的无花果

`figo` 在意大利语 / 葡萄牙语里就是**无花果**。标记画的是它的**剖面**——
而且配色不是硬套的:无花果天生就是「紫皮 + 粉红果肉」,**logo 即品牌色的实体化**。

- **紫色表皮**(`#6D54F2`) = 外壳 / 运行时——把设计包起来跑
- **粉红果肉**(`#F0476B → #FF8A4C`) = 里面承载的设计
- **密布的籽** = 一个个组件 / 节点
- **顶上的叶** = 还活着、会生长

字标里把 **"o" 直接做成无花果剖面**——`figo` 谐音 fig + o,那个 o 就是 fig。
小尺寸(favicon / app 图标)用**实心白果**版,把细节让位给辨识度。

## 配色

| token | hex | 含义 |
|---|---|---|
| `--brand-warm`  | `#FF8A4C` | 暖珊瑚 · 果心 |
| `--brand-mid`   | `#F0476B` | 品红 · 果肉 |
| `--brand-cool`  | `#6D54F2` | 紫 · 表皮 / 运行时 |
| `--brand-leaf`  | `#56C06E` | 绿 · 叶 |
| `--brand-ink`   | `#16161E` | 字标(浅底) |
| `--brand-paper` | `#14141C` | 深底背景 |

主渐变(用于实心版 / app 图标):
`linear-gradient(135deg, #FF8A4C 0%, #F0476B 50%, #6D54F2 100%)`。
果肉径向渐变:中心 `#FFD2AC` → `#F4647F` → 边缘 `#E0426C`。

## 文件

| 文件 | 用途 |
|---|---|
| `figo-mark.svg`      | 主标记,无花果剖面(彩色),透明底。≥32px 场景、文档插图 |
| `figo-app-icon.svg`  | 实心白果 + 渐变方块底。app 图标 / favicon / 小尺寸(16px 仍清晰) |
| `figo-logo.svg`      | 字标——**"o" 就是无花果剖面**(浅底) |
| `figo-logo-dark.svg` | 字标——同上(深底) |

全部为矢量 SVG。需要 PNG:`qlmanage -t -s 512 -o . figo-app-icon.svg`(macOS),
或任意 SVG→PNG 工具。

## 使用规范

- **小尺寸用实心版**:剖面的籽/果肉在 ≤16px 会糊,favicon/小图标一律用 `figo-app-icon.svg`。
- **留白**:logo 四周至少留出半颗果的高度作安全边距。
- **别改**:果形、籽的分布、渐变方向、字标字距。单色场景用实心果形
  (`figo-app-icon.svg` 的白果)或整体填 `#6D54F2`。
- **字标字体**:wordmark 当前用 Helvetica/Arial Bold(字距 -3)。
  对外正式分发前,建议把 `figo` 文本转成轮廓路径(outline),去掉字体依赖。
